# 分解原理 (Decompose Principle)

## 核心框架

```
根节点 ────────> 第一性原理（基向量）
     │
     ├─ 压缩 ─────> 降维：现象空间 → 原理空间
     │              找到最底层的基本原理
     │
     └─ 泛化 ─────> 向量组合：新实例 = Σ(基向量 × 权重)
                    用原理解释/预测新现象
```

**核心思想：**
- 所有复杂知识都可分解为少数"基向量"（第一性原理）
- 压缩 = 从现象中提取根节点
- 泛化 = 用根节点组合产生新理解

---

## 使用场景

### 1. 概念分析
将复杂概念分解到底层原理

**示例请求：**
- "分解'商业模式'这个概念"
- "什么是'系统思维'的根节点？"

### 2. 问题求解
将复杂问题投影到原理空间

**示例请求：**
- "帮我用分解原理解决用户留存问题"
- "分析为什么这个产品会失败"

### 3. 创新思维
跨领域原理组合，生成新想法

**示例请求：**
- "结合生物学和工程学原理，设计新型材料"
- "用分解原理想出一个新的商业模式"

### 4. 知识管理
压缩知识点到根节点，建立可泛化的知识体系

**示例请求：**
- "帮我整理营销学知识体系"
- "将这个领域压缩成5个核心原理"

## 输出形式

**选择原则：**
- 概念分析/知识管理 → 结构化分析 + 向量公式
- 问题求解 → 向量公式 + 对话式引导
- 创新思维 → 向量公式 + 思维导图
- 复杂深度分析 → 组合所有四种形式

---

### 1. 结构化分析
以树状/表格形式展示：
```
概念：[名称]
├─ 根节点（基向量）
│  ├─ 原理1：[名称 + 解释]
│  ├─ 原理2：[名称 + 解释]
│  └─ 原理3：[名称 + 解释]
├─ 压缩过程：现象 → 原理的映射关系
└─ 泛化方向：可用于解释的新领域/问题
```

### 2. 向量公式
用数学化表示：
```
概念A = α·原理1 + β·原理2 + γ·原理3

类比推理：
国王 - 男 + 女 = 女王

创新推理：
原理A + 原理B - 约束C = 新事物
```

### 3. 思维导图
可视化的原理空间映射：
```
        原理空间
       /    |    \
   原理1  原理2  原理3（基向量）
     \     |     /
      \    |    /
      现象/问题（在空间中的位置）
```

### 4. 对话式引导
通过问题引导用户自己发现原理：
```
1. 这个概念的"最小可分单元"是什么？
2. 哪些原理是"不可或缺的"？
3. 如果去掉这个概念，世界会如何改变？
```

## 工作流程

### 第一步：识别分析对象
明确用户请求属于哪个场景（概念分析/问题求解/创新/知识管理）

### 第二步：根节点提取
**核心问题：**"这个概念/问题的最小、最基础的组成部分是什么？"

**提取方法：**

1. **剥离法**（层层追问）
   ```
   概念 → 为什么？ → 更深层原理 → 为什么？ → ... → 根节点
   示例："商业模式" → 为什么存在？ → 创造价值 → 如何持续？ → 获取价值
   ```
   - 每一层都问"这个概念基于什么？"
   - 直到到达无法继续分解的基础原理
   - 通常需要3-5层追问

2. **反证法**（测试必要性）
   ```
   测试：去掉某元素，概念是否崩塌？
   示例：去掉"价值创造"，商业模式还成立吗？不成立 → 根节点
   ```
   - 逐一测试各个组成部分
   - 如果去掉后概念不复存在，则是根节点
   - 适合验证已识别的根节点

3. **跨域验证**（寻找共性）
   ```
   多领域对比 → 找共同模式 → 提取原理
   示例：生物/物理/经济都有"能量" → 能量是通用基向量
   ```
   - 在不同领域寻找相同的底层模式
   - 跨域的共同点通常是更深层的原理
   - 适合发现创新连接点

**数量原则：**
- 简单概念：3-5个根节点
- 复杂概念：5-7个根节点
- 超过7个：可能需要进一步压缩

### 第三步：压缩映射
建立现象到原理的映射：
```
具体现象 → 抽象 → 原理表示
```

### 第四步：泛化应用
用原理组合产生新理解或解决方案

### 第五步：输出呈现
根据请求自动选择或组合四种输出形式

## 示例：词向量的分解

**概念：** 词向量空间

**根节点（基向量）：**
1. **语义维度**：词义的本质（如"性别"、"统治"、"血缘"）
2. **关系维度**：词与词之间的结构关系（如"是-一种"、"整体-部分"）
3. **上下文维度**：词的使用场景和语境

**压缩过程：**
```
"国王" = 统治者 + 男性 + 贵族
"女王" = 统治者 + 女性 + 贵族

词向量空间将数百万个词压缩到语义基向量的组合
```

**泛化应用：**
```
类比推理：
国王 - 男 + 女 = 女王
王子 - 男 + 女 = 公主

创新推理：
AI + 创造力 = 创造性AI
```

## 关键启发式

### 压缩启发式
- **奥卡姆向量版**：用最少基向量解释最多现象
- **5个基向量原则**：大多数复杂概念可压缩到3-7个核心原理
- **跨域对称性**：不同领域往往共享相同的基向量

### 泛化启发式
- **类比即向量运算**：类比推理 = 原理向量的加减
- **创新即向量组合**：创新 = 新的基向量组合方式
- **预测即向量投影**：用已知原理预测未知现象

### 验证启发式
- **解释力测试**：原理能否解释80%以上相关现象？
- **简洁性测试**：是否存在更少的基向量也能解释？
- **一致性测试**：原理之间是否存在矛盾？

## 调试与迭代

**如果分解效果不佳，检查：**

1. **根节点是否真的"根"？**
   - 继续追问"这个原理本身又基于什么？"

2. **压缩是否过度？**
   - 是否丢失了关键维度？考虑增加基向量

3. **泛化能力如何？**
   - 用原理解释一个新场景，看是否可行

4. **输出形式是否匹配需求？**
   - 调整结构化/公式/导图/对话的比例
